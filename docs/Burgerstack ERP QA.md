### 입고 관리

#### 입고 예정 목록 페이지

- 날짜 필터링 구현
- 검색어 필터링 구현
- 총액, 요청일 전체 일관성 기준으로 포맷팅

#### 입고 처리

- 초기화 버튼 옆에 목록 버튼(입고 예정 목록으로 이동) 추가
- 필터링 기능은 완전히 제거
- `정상 입고 수량` -> `실입고 수량`으로 이름 변경
- `불량 수량` -> `차이 수량`으로 이름 변경
- `불량사유` -> `사유`로 이름 변경
- 이상 수량 컬럼은 입력 불가능 -> `실입고 수량 - 승인 수량` 수식으로 자동으로 계산
- `금액` -> `차액`으로 이름 변경
- 아래쪽 `총 금액`-> `총계`, `￦` 특수문자, 아예 지워버리기 중 택1
- 저장 버튼 클릭 시 확인 모달창 표시
	- 모달창에는 차이 수량이 존재하는 행을 요약(차이 수량, 사유, 총계)해서 출력
	- 모달 안에 비고(receiptMemo) textarea 추가 -> DB에는 `RECEIPT_MEMO`에 입력

#### 입고 이력 목록

- 컬럼을 `입고 코드`, `품목요약`, `상태`,` 입고 완료일시`로 고정
- `발주 번호` 컬럼 제거
- `품목 요약` 컬럼 추가
- `입고 메모` 컬럼을 `상태` 컬럼으로 이름 변경
- `상태` 컬럼은 DB에서 각 상세 품목의 상태를 계산해서 출력할 것
- DB의 `RECEIPT_MEMO` 컬럼은 점주가 입고 처리 시에 적는 비고를 저장

#### 입고 이력 상세보기

- 정보
	- `입고 코드`, `발주 코드`, `입고 상태`, `입고 완료일시`로 고정
	- `입고 메모`는 `비고`로 이름 변경
	- `입고 상태`는 DB에서 각 상세 품목의 상태를 계산해서 출력할 것
	- `비고`는 표 아래쪽에 추가
- 표
	- `자재 코드`, `자재명`, `자재 유형`, `단가`, `승인 수량`, `요청 수량`, `실입고 수량`, `차이 수량`, `사유`, `금액`으로 고정
	- `실수령 수량` -> `실입고 수량`으로 이름 변경
	- `불량 수량` -> `차이 수량`으로 이름 변경
	- `불량사유` -> `사유`로 이름 변경
	- 금액, 단가는 전체 일관성 기준으로 포맷팅
	- 자재 유형은 타입 코드가 아니라 한글로 표기되도록 변경
